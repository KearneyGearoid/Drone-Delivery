Drone Delivery Service

Brief Use Case 6: Create Group Delivery

The requester initiates this by selecting the create group delivery button. For each delivery location the requester will be asked to enter the weight of package, dimensions of package, location of delivery, location of collection. The system validates this and assigns a drone to the delivery. The requesters dashboard is updated with the new delivery.

Fully Dressed Use Case 6: Create Group Delivery

Use Case Name: Create Group Delivery
Priority: 3
Source: Registered Users
Primary Business Actor: Registered Users
Goal in Context: For Registered Users to create a group delivery through the system
Description: This use case will allow registered users to create a group delivery

Stakeholders and Interests:

Registered User - Wants to be able to create a group delivery with another registered user.

Pre Conditions:Both Users Must Be Registered

Post Conditions: None

Typical Scenario:
  1. Registered User: Invokes Create Group Delivery Function
  2. Delivery System: System retrieves details that need to be entered and displays on UI
  3. Registered User: User enters new group delivery details:
    • Names of Registered Users
    • Locations of Delivery
    • Locations of Collection
    • Phone Numbers of Registered Users
    • Delivery ID
    • Delivery Name
    • Packages Weights
    • Packages Dimensions
    • Delivery Cost
  4. Registered User: User confirms deliverys to be created
  5. Delivery System: System Validates data entered:
    • All fields must be entered
    • Package Weights and Dimensions must be accurate
  6. Delivery System: System assignes correct drone to delivery
  7. Registered User: User recieves confirmation message that delivery was successful
  8. Delivery System: System Database is updated with new group delivery

Conclusions: A new Group Delivery is added
Post Conditions: A Group Delivery will be made from a certain location and delivered to a certain location.
