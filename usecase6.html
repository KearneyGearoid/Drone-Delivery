Drone Delivery Service

Brief Use Case 6: Create Group Delivery

The requester initiates this by selecting the create group delivery button. For each delivery location the requester will be asked to enter the weight of package, dimensions of package, location of delivery, location of collection. The system validates this and assigns a drone to the delivery. The requesters dashboard is updated with the new delivery.

Fully Dressed Use Case 6: Create Group Delivery

Use Case Name: Create Group Delivery
Priority: 3
Source: Users
Primary Business Actor: Users
Goal in Context: Users create a group delivery through the system Description: This use case will allow users to create a group delivery
Typical Scenario:

Actor Action System Response
Step 1: User invokes Create Group Delivery Function Step 2: System retrieves details that need to be entered and displays on UI
Step 3: User enters new group delivery details:
  • Names of Users
  • Locations of Delivery
  • Locations of Collection
  • Phone Numbers
  • Packages ID
  • Packages Names
  • Packages Weights
  • Packages Dimensions
Step 4: User confirms deliverys to be created Step 5: System Validates data entered:
  • All fields must be entered
  • Package Weights and Dimensions must be accurate
 Step 6: System assignes correct drone to delivery Step 7: System displays conformation message Step 8: System Database is updated with new group delivery Step 9: System clears UI

Conclusions: A new Group Delivery is added
Post Conditions: A Group Delivery will be made from a certain location and delivered to a certain location.
