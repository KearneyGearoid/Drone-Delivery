Drone Delivery Service

Brief Use Case 8: Track Delivery

This use case allows the user to track their delivery

When the user wishes to track their delivery, they will open the Track Delivery method and enter in the unigue identifier code that they are supplied. Form here the system will search the database for their package and return where the package is and what depos it has passed through allready.

Fully Dressed Use Case 8: Track Delivery

Actors: Registered User, Event System

Goal in Context: Registered users be able to track a delivery made with the system

Scope: A drone delivery system

Level: User goal

Stakeholders and Interests:

Registered User - Wants to be able to view delivery and track progress.

Pre Conditions:User has a delivery on the system. Database is accessible

Post Conditions: None

Success Scenario:

  1. Registered User: Clicks the ‘Track Delivery’ button.
  2. Registered User: Inputs unique delivery code details.
  3. Delivery System: Validates user input.
  4. Delivery System: System gets information on the delivery
  5. Delivery System: System tracks the last location of drone
  6. Delivery System: System estimates time of arrival based on current location and delivery address
  7. Delivery System: System displays all current information on the delivery

Extensions:

  • 2a. The user does not know the unique tracking code
